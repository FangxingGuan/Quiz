**状态**: 草案

## 一、产品概述

### 1.1 背景

在新闻资讯类产品中，用户留存和内容消费深度是核心指标。互动测试（Quiz）作为轻量级互动形式，具备以下价值：

- **提升用户活跃度**：趣味测试降低参与门槛，增加用户停留时长
- **驱动社交裂变**：测试结果天然具有分享属性，带来免费流量
- **沉淀用户画像**：通过答题行为获取显性兴趣标签，优化推荐算法
- **连接内容消费**：测试结果页关联推荐新闻，形成"测试 → 阅读"闭环

### 1.2 目标

| 指标 | 目标 |
|------|------|
| 测试完成率 | > 70% |
| 结果页分享率 | > 15% |
| 分享回流率 | > 8% |
| 结果页新闻点击率 | > 20% |
| 用户标签覆盖率 | 首月覆盖 30% DAU |

### 1.3 用户路径

```
入口（首页卡片/推送/分享链接）
  → 测试介绍页（封面 + 开始按钮）
  → 答题页（5-10 题，< 2 分钟）
  → 结果页（结果展示 + 3-5 条相关新闻 + 分享按钮）
  → 分享（海报/卡片 → 社交平台 → 带链接回流）
  → 数据沉淀（兴趣标签写入用户画像）
```

## 三、测试详细设计

### 3.1 性格/心理类

#### #1 MBTI 性格测试 `P0`

| 项目 | 内容 |
|------|------|
| **玩法** | 8-10 道情景选择题，测出 16 种性格类型（如 INFP / ENTJ） |
| **题目示例** | "周末你更喜欢？A. 和朋友聚会 B. 独自阅读" |
| **结果输出** | 性格类型 + 一句话描述 + 代表人物 + 优劣势简析 |
| **推荐新闻** | 职场、心理学、生活方式类文章 |
| **分享形式** | 性格类型结果卡片 + 趣味标题（如"原来你是 INFP 型调解者！"） |
| **标签沉淀** | `personality:mbti_type`, `interest:psychology`, `style:introvert/extrovert` |

#### #2 大五人格指数 `P2`

| 项目 | 内容 |
|------|------|
| **玩法** | 10 道量表题（1-5 分），测出五维度得分 |
| **五维度** | 外向性、开放性、责任心、宜人性、神经质 |
| **结果输出** | 雷达图/柱状图 + 各维度简析 |
| **推荐新闻** | 心理学、个人成长、健康类文章 |
| **分享形式** | 五维雷达图结果海报 |

## 四、技术架构概要

### 4.1 系统模块

```
┌─────────────────────────────────────────────────┐
│                  前端展示层                        │
│  测试入口 → 答题页 → 结果页 → 分享组件             │
├─────────────────────────────────────────────────┤
│                  业务逻辑层                        │
│  题库管理 │ 评分引擎 │ 结果生成 │ 分享追踪          │
├─────────────────────────────────────────────────┤
│                  数据层                            │
│  题库 DB │ 用户答题记录 │ 标签系统 │ 推荐引擎对接    │
└─────────────────────────────────────────────────┘
```

### 4.2 核心数据模型

```
Quiz（测试）
├── quiz_id, title, category, status
├── questions[]
│   ├── question_id, text, type(single/multi/scale)
│   └── options[]
│       ├── option_id, text, image_url
│       └── tag_weights{}  ← 每个选项对标签的贡献权重
├── results[]
│   ├── result_id, title, description, image_url
│   ├── condition{}  ← 触发条件（分数区间/标签匹配）
│   └── recommended_news_tags[]
└── share_template{}

UserQuizRecord（用户答题记录）
├── user_id, quiz_id, timestamp
├── answers[]
├── result_id
├── shared: boolean
└── generated_tags[]

UserProfile（用户画像扩展）
├── user_id
├── interest_tags{}  ← 累积标签权重
├── personality_tags{}
└── quiz_history[]
```

### 4.3 关键技术点

| 模块 | 方案 |
|------|------|
| 分享海报生成 | 服务端 Canvas 渲染 / 前端 html2canvas |
| 分享追踪 | 唯一短链 + UTM 参数 |
| 标签写入 | 异步消息队列 → 用户画像服务 |
| 题库管理 | 运营后台 CMS，支持动态上下线 |
| A/B 测试 | 支持同一测试多版本题目/结果 |
